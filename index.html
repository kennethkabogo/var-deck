The Problem
Agents act.
Nobody can prove what they did.
When an AI agent executes a financial transaction, routes a payment, or makes a compliance-critical decision — the only record is a software log. Software logs can be altered. Any compromised host can silently rewrite them. There is no cryptographic proof of what the agent decided, what data it saw, or whether the binary running it was tampered with.
Live Evidence — April 18, 2026

KelpDAO lost $290M because their verifier had no hardware-level execution boundary. Lazarus Group swapped binaries on the RPC nodes feeding the verifier. The malicious nodes showed truth to monitors, lies to the verifier. No attestation. No recourse. In a VAR architecture this attack fails at step one — a binary swap changes the PCR0 measurement, the KMS policy gate closes, and nothing settles.

Pain Point 01
Institutions won't let agents touch capital
No cryptographic proof of agent behavior means no institutional trust. Human backstops eliminate the efficiency gain that made agents worth building.
Pain Point 02
Compliance can't survive regulatory review
When a regulator asks "what did the agent do before it moved that capital?" — a log file is not a sufficient answer. The evidence needs to be hardware-rooted and tamper-evident.
Pain Point 03
Dispute resolution is slow and manual
Reconstructing what an agent did from logs takes weeks. An APEX bundle reduces that to hours — the proof already exists, produced in real time during execution.
The Solution
VAR — Verifiable Agent Runtime
VAR wraps any AI agent in a hardware-attested enclave. Every action produces a cryptographic receipt in real time — hardware-signed, PCR-bound, independently verifiable by any third party without trusting the operator or the host. The proof is produced during execution, not reconstructed afterward. By the time the agent finishes, the evidence already exists.

The Stack
VAR proves it.
Finfiti settles it.
VAR is the attestation primitive. Finfiti is the programmable settlement interface that lets agents touch money — route payments, release escrow, settle transactions — against a verified APEX attestation. Its value is derived entirely from VAR. Without the attestation layer, it's another fintech API. With it, it's the primitive that makes agent-native finance possible.
The S3 analogy

S3 is just object storage — a simple idea. But because it sits beneath everything, it became one of the most valuable pieces of infrastructure on earth. Finfiti is settlement storage: the lowest-level interface between a verified agent action and a financial consequence. We built it ourselves as the first application on APEX. It proved the primitive works in production. That's its job.
